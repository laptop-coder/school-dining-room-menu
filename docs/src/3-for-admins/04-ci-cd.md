# CI/CD
## Как это работает?
На GitHub есть две ветки: `dev` и `main`. Разработка происходит в ветке `dev`. Когда функция готова, ветка `dev` сливается в `main`. При попадании в `main` код автоматически собирается и выкладывается на Docker Hub. Расписание в `cron` настроено таким образом, что каждый день в 05:00 UTC+03:00 проект полностью перезапускается на сервере.
## Как использовать?
Все изменения в коде, которые сегодня были сделаны и отправлены, на следующее утро будут у пользователя. С одной стороны, проект всё время будет поддержан в актуальном состоянии, а с другой стороны увеличивается риск возникновения ошибок.
